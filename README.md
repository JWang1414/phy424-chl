These are a few notes to help me remember the organization of the data I've saved here.

### SIN

I have one test sample here. The file names indicate the "period" setting I have used for the SLM

### BLAZED

All samples here have been taken with a period of 64. This is just because I found the samples look best with this number.

### STRIPES, SOLID, CHECKERBOARD

For some samples I have indicated "pps" the "pixels per stripe" This is a measure of the period of the stripes, or checkboard squares, or whatever. Smaller pps correspond with a higher frequency, just like with gratings.

### STRIPES, CHECKERBOARD

The file names here indicate the "pixel value one" followed by "pixel value two". That is, they indicate the phase difference in the SLM.

(Pixel value one)-(Pixel value two)

### SOLID

There is only one value you can adjust here, pixel value one. Labelled by the file name.

### RANDOM

You cannot adjust anything, the file names here are simple the trial number. For later trials, I progressively increased the exposure time more and more.

### Fresnel

The resulting beam when reflected from the Fresnel lens pattern. I specifically used a radius of 200px, and defocus 5.0. After roughly locating the waist, I took images offset from the waist, the distance offset is the file names.

### Rayleigh Range

Rough measurements of the spot size from the Fresnel lens pattern. The file name indicate the camera's location on the moving table, in cm. The possible waists include a number of trials I took to locate the waist as accurately as possible. These images were analyzed with ImageJ.

### Arbitrary

Files I used to help generate an arbitrary image using the SLM
